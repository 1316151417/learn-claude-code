# Deep Research Agent 设计文档

## 1. 概述

### 1.1 项目定位
Deep Research Agent 是一个基于 Claude Agent SDK 的极简深度研究代理，旨在通过自动化工具链完成复杂主题的深度研究与分析。

### 1.2 核心价值
- **极简设计**：仅约 60 行代码，实现完整的研究代理功能
- **自动化工具链**：自动使用 Claude Code CLI 的配置和工具集
- **深度分析能力**：不满足表面答案，挖掘底层原理
- **结构化输出**：提供清晰、有组织的研究结果

### 1.3 技术栈
- **运行时**：Python 3.8+
- **异步框架**：anyio
- **AI SDK**：claude_agent_sdk
- **工具集**：继承 Claude Code CLI 完整工具集（bash, read, web_search 等）

## 2. 设计思路

### 2.1 核心设计原则

#### 2.1.1 极简主义
> **Less is More**: 代码的 20% 提供 80% 的功能

通过最小化代码实现最大化功能：
- 复用 Claude Code CLI 的配置和工具
- 利用 Claude 模型本身的研究能力
- 减少中间层，直接调用 SDK

#### 2.1.2 智能代理
> **Model is Agent**: 模型本身就是代理，代码只是提供工具

设计哲学：
- 模型负责"做什么"（决策）
- SDK 负责"怎么做"（执行）
- 代码负责"用什么"（工具）

#### 2.1.3 自动化优先
> **Zero Configuration**: 开箱即用

- 自动读取 Claude Code 配置
- 自动加载完整工具集
- 自动处理多轮对话

### 2.2 架构设计

```
┌─────────────────────────────────────────────────┐
│               用户交互层                          │
│          (main: REPL循环)                        │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              业务逻辑层                           │
│        (deep_research: 研究协调)                 │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              SDK 抽象层                           │
│     (query: 流式查询 + 工具调用)                  │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              Claude 模型层                        │
│  (Claude Sonnet 4.5: 推理 + 工具使用)            │
└─────────────────────────────────────────────────┘
```

### 2.3 关键设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 异步框架 | anyio | 更好的异步兼容性，支持多种后端 |
| 消息处理 | 流式 (async for) | 实时反馈，更好的用户体验 |
| 工具限制 | 不限制 | 让模型自主选择最合适的工具 |
| 输出格式 | 混合打印 | 统一显示文本和工具调用信息 |

## 3. 模块划分

### 3.1 模块结构图

```
deep_research.py
├── 配置层
│   └── ClaudeAgentOptions (系统提示 + 工具配置)
├── 核心层
│   ├── deep_research() (异步研究函数)
│   └── query() (SDK 流式查询)
├── 交互层
│   └── main() (REPL 循环)
└── 辅助层

```

### 3.2 详细模块说明

#### 3.2.1 配置模块

**职责**：定义代理的行为和能力

```python
options = ClaudeAgentOptions(
    system_prompt=system_prompt,
    # 不限制工具，让 Claude 自主选择
)
```

**系统提示结构**：
- **核心能力**：深入分析、多源验证、结构输出
- **研究流程**：理解 → 分解 → 搜索 → 综合
- **工具使用**：web_search、bash、read
- **输出格式**：核心结论 + 详细分析 + 延伸阅读

#### 3.2.2 核心模块

**deep_research() 函数**：
- **输入**：研究主题 (topic: str)
- **输出**：流式消息 (AsyncGenerator)
- **职责**：
  1. 构建完整提示
  2. 调用 SDK 查询
  3. 处理流式响应
  4. 格式化输出

**消息类型处理**：
```python
if msg_type == "text":
    # 流式文本输出
elif msg_type == "tool_use":
    # 显示工具调用
elif msg_type == "tool_result":
    # 显示工具执行结果
```

#### 3.2.3 交互模块

**main() 函数**：
- **模式**：REPL (Read-Eval-Print Loop)
- **特性**：
  - 欢迎界面 + 使用说明
  - 循环读取用户输入
  - 优雅退出处理
  - 异常捕获与错误提示

#### 3.2.4 辅助模块

**消息解析器**：
```python
if isinstance(message, dict):
    msg_type = message.get("type", "")
    content = message.get("content", "")
    # 根据类型分发处理
```

**错误处理器**：
```python
try:
    anyio.run(deep_research, topic)
except Exception as e:
    print(f"\\n❌ Error: {e}")
```

#### 4.2.2 消息处理流程（代码侧）

```python
# 伪代码表示
async for message in query(prompt, options):
    match message.type:
        case "text":
            print(content, end="", flush=True)  # 流式输出
        case "tool_use":
            print(f"> 使用工具: {name} {input}")
            # SDK 自动执行工具
        case "tool_result":
            print("✓ 工具执行完成")
            # 结果自动添加到上下文
        case _:
            print(message)  # 兜底处理
```

## 5. 技术要点

### 5.1 异步编程

**为什么使用 anyio？**
- 支持多种异步后端（asyncio, trio）
- 更好的异步原语抽象
- 与 claude_agent_sdk 兼容

**流式处理优势**：
- 实时反馈，提升用户体验
- 降低内存占用
- 支持长响应场景

### 5.2 消息类型系统

```python
message = {
    "type": "text" | "tool_use" | "tool_result",
    "content": str | dict
}
```

| 类型 | content 类型 | 处理方式 |
|------|-------------|---------|
| text | str | 直接流式输出 |
| tool_use | dict (name, input) | 显示工具信息 |
| tool_result | dict | 显示完成状态 |

### 5.3 系统提示工程

**提示设计原则**：
1. **角色定义**：明确"你是深度研究代理"
2. **能力声明**：列出核心能力
3. **流程指导**：提供研究步骤
4. **格式约束**：指定输出结构
5. **工具引导**：建议使用哪些工具

**提示效果**：
- 引导模型进行深度分析
- 确保结构化输出
- 提升研究质量

### 5.4 工具生态

**继承的工具集**（来自 Claude Code CLI）：
- `web_search`: 网络搜索
- `read`: 读取文件
- `bash`: 执行命令
- `write`: 写入文件
- `edit`: 编辑文件
- 更多...

**不限制工具的优势**：
- 模型自主决策
- 灵活应对不同场景
- 未来工具自动可用

## 8. 总结

Deep Research Agent 展示了如何用最少代码实现功能完整的 AI 研究代理：

### 8.1 核心优势
- **极简**：60 行代码，完整功能
- **强大**：依托 Claude 的强大能力
- **灵活**：自动继承工具和配置
- **可扩展**：易于定制和扩展

### 8.2 设计精髓
> 最好的代码是"没有代码"——让 AI 做 AI 擅长的事，让代码只做必须的事。

### 8.3 适用场景
- 技术调研
- 学术研究
- 市场分析
- 竞品分析
- 深度学习

### 8.4 局限性
- 依赖 Claude API（需要网络）
- 成本考虑（API 调用费用）
- 结果质量取决于模型能力
