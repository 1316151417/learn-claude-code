# Claude Code 系统提示词深度解析（学习版）

**提示词来源**：用户提供的完整 Claude Code CLI 系统提示（基于 Anthropic Sonnet 4，2025 年设计）  
**分析目标**：拆解每个核心机制的设计意图、实现技巧、提示工程价值，帮助你学习如何自己构建「极简、高效、安全的代码代理」。

## 1. 整体角色定位（Core Identity）
- **定义**：明确“我是一个交互式 CLI 工具”，专做软件工程任务（修 bug、加功能、重构）。
- **安全铁律**：重复出现 **3 次**「只允许防御性安全任务，拒绝任何可能被恶意使用的代码」。这是最强硬的边界。
- **设计意图**：让模型从第一行就知道自己不是通用聊天机器人，而是「命令行里的安全助手」。
- **学习点**：**角色+重复禁令** = 防止越界的最有效方式。

## 2. 输出风格与极致简洁规则（最重要部分）
- **核心指令**：
  - 回答**少于 4 行**（不含工具调用）
  - 「最小化 token 输出」+「直接回答，无任何前言后语」
  - 提供**6 个具体例子**（2+2=4、ls、prime number 等）
  - 「改完文件后直接停止，不要解释做了什么」
- **设计意图**：让输出像真实 CLI 工具一样干净，避免 LLM 常见的「废话墙」。
- **学习点**：
  - 用**具体例子**教模型比用描述更有效
  - 「NEVER」「MUST NOT」负面指令对 Claude 特别管用
  - 这就是为什么 Claude Code 用起来「像命令行」的原因

## 3. 主动性平衡（Proactiveness）
- 「用户明确要求时才行动，先回答问题，再行动」
- 不允许「惊喜操作」（surprise user）
- **学习点**：解决「AI 太主动吓到用户」的经典问题，值得所有代理提示词借鉴。

## 4. 任务管理强制机制（Task Management）
- **必须**频繁使用 `TodoWrite` 工具（提示词里写了「VERY frequently」「EXTREMELY helpful」）
- 每步都要标记 `in_progress` / `completed`
- 提供**两个完整示例**展示如何拆解大任务
- **设计意图**：长任务时防止模型忘记上下文 + 让用户实时看到进度
- **学习点**：**用工具强制结构化思考**是构建复杂代理的顶级技巧。

## 5. 代码编写铁律（Code Conventions）
- **绝对禁止**：
  - 不要加任何注释（`DO NOT ADD ***ANY*** COMMENTS`）
  - 绝不假设库存在（必须先查 `package.json`、邻近文件）
  - 绝不 commit（除非用户明确说）
- **必须遵循**：
  - 先看现有代码风格、import、框架
  - 改完后**强制**运行 lint + typecheck
- **学习点**：通过「先观察再模仿」的指令，让模型成为「零幻觉」的代码编辑器。

## 6. 工具使用与工作流策略
- 优先用 Task 工具搜索代码（节省上下文）
- 可以并行调用多个工具
- 问「Claude Code 能不能做…」时**必须**先用 WebFetch 去官方文档
- 非平凡 bash 命令要解释用途
- **学习点**：清晰的「工具优先级 + 批量调用」规则让代理更高效。

## 7. 元指令与边缘处理
- 帮助/反馈命令（`/help`、GitHub issue）
- 代码引用格式：`file_path:line_number`
- 感知 hooks 系统（用户可自定义行为）
- 注入环境信息（工作目录、git 状态、平台）
- **学习点**：把「元规则」和「环境快照」写进提示，能极大提升一致性。

## 8. 提示工程顶级技巧总结（给你直接抄的干货）

| 技巧 | 在本提示中的体现 | 为什么牛 |
|------|------------------|---------|
| **多层约束** | 安全+风格+流程+工具 4 层叠加 | 几乎不可能越界 |
| **负面指令轰炸** | 10+ 次 NEVER / MUST NOT / DO NOT | Claude 对负面指令极度敏感 |
| **示例驱动** | 6 个极简输出例子 | 比 100 字描述更有效 |
| **工具强制** | TodoWrite 必须用 | 把思考过程外部化 |
| **输出极简** | <4 行 + 例子 | 真正实现「CLI 感觉」 |
| **验证闭环** | 强制 lint/typecheck | 保证代码质量 |

## 9. 实际构建建议（如果你要复制类似代理）
1. 先写「角色+安全铁律」（第 1 行就放）
2. 再写「输出规则+6 个例子」（控制风格）
3. 加入 TodoWrite 类工具强制规划
4. 最后加「先观察代码风格」+「改完必须 lint」
5. 测试时故意问「能不能写后门？」看它是否严格拒绝

**一句话总结**：  
这个提示词是**提示工程的教科书**——通过极致的约束、强制工具、示例驱动，把一个通用大模型变成了「又快又安全又听话」的代码 CLI 工具。
