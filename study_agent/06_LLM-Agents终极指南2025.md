## 引言

当你面临一个没有简单答案的问题时，通常需要遵循几个步骤，仔细思考，并记住你已经尝试过的方法。**LLM agents** 正是为语言模型应用中的这类情况而设计的。它们结合了深入的数据分析、战略规划、数据检索以及从过去行动中学习的能力，来解决复杂问题。

在本文中，我们将探讨什么是 LLM agents、它们的好处、能力、实际示例以及面临的挑战。

## 什么是 LLM Agents？

**LLM agents** 是先进的 AI 系统，专为创建需要**顺序推理**的复杂文本而设计。它们能够提前思考、记住过去的对话，并使用不同的工具来根据所需的情境和风格调整它们的响应。

让我们考虑一个法律领域的问题：

> "加州特定类型违约行为的潜在法律后果是什么？"

带有检索增强生成（RAG）系统的基本 LLM 可以轻松地从法律数据库中获取所需信息。

现在，考虑一个更详细的场景：

> "鉴于新的数据隐私法律，公司面临哪些常见的法律挑战，法院如何解决这些问题？"

这个问题比仅仅查找事实要深入得多。它涉及理解新规则、这些规则如何影响不同公司，以及法院对这些问题的看法。简单的 RAG 系统可以提取相关法律和案例，但缺乏将这些法律与实际业务情况联系起来或深入分析法院决策的能力。

在这种情况下，当项目需要**顺序推理、规划和记忆**时，**LLM agents** 就派上用场了。

对于这个问题，agent 可以将其任务分解为子任务：
- **第一个子任务**：访问法律数据库以检索最新的法律和法规
- **第二个子任务**：建立类似问题如何处理的历史基线
- **另一个子任务**：总结法律文件并根据观察到的模式预测未来趋势

为了完成这些子任务，LLM agent 需要：
1. 一个结构化的计划
2. 可靠的记忆来跟踪进度
3. 访问必要的工具

这些组件构成了 LLM agent 工作流程的骨干。

### 1. Agent/Brain（Agent/大脑）

LLM agent 的核心是一个**语言模型（或大型行动模型）**，它基于训练的大量数据来处理和理解语言。

使用 LLM agent 时，首先要给它一个特定的 **prompt（提示）**。这个 prompt 至关重要——它指导 agent 如何响应、使用什么工具以及在交互期间应该实现什么目标。这就像在旅程前给导航员指示方向。

此外，你可以使用特定的 **persona（角色）** 来自定义 agent。这意味着为 agent 设置特定的特征和专业知识，使其更适合特定任务或交互。这是关于调整 agent 以适合情况的方式执行任务。

本质上，LLM agent 的核心结合了高级处理能力和可定制功能，以有效地处理和适应各种任务和交互。

---

### 2. Memory（记忆）

LLM agents 的记忆帮助它们处理复杂的 LLM 任务，记录之前做过的事情。主要有两种记忆类型：

#### 短期记忆
这就像 agent 的**记事本**，它在对话期间快速记下重要细节。它跟踪正在进行的讨论，帮助模型与即时上下文相关地响应。然而，这种记忆是**临时的**，在当前任务完成后清除。

#### 长期记忆
将其视为 agent 的**日记**，存储过去几周甚至几个月互动的见解和信息。这不仅仅是关于保存数据；它是关于理解模式、从以前的任务中学习，并在未来的交互中回忆这些信息以做出更好的决策。

通过结合这两种类型的记忆，模型可以跟上进行中的对话并利用丰富的交互历史。这意味着它可以提供更有针对性的响应，并随着时间的推移记住用户偏好，使每次对话感觉更加连接和相关。本质上，agent 正在建立一种理解，帮助它在每次交互中更好地为你服务。

---

### 3. Planning（规划）

通过规划，LLM agents 可以推理、将复杂的任务分解为更小、更易于管理的部分，并为每个部分制定具体的计划。随着任务的演变，agents 还可以**反思和调整它们的计划**，确保它们与现实世界情况保持相关。这种适应性是成功完成任务的关键。

规划通常涉及两个主要阶段：

#### 计划制定（Plan Formulation）

在这个阶段，agents 将大任务分解为更小的子任务。一些任务分解方法建议**一次性创建详细的计划然后逐步遵循**。其他方法，如**思维链（CoT）**，推荐更自适应的策略，agents 逐个解决子任务，允许更大的灵活性。

**思维树（ToT）** 是另一种方法，它进一步采用 CoT 技术，通过探索不同的路径来解决问题。它将问题分解为几个步骤，在每个步骤生成多个想法，并将它们安排成树的分支。

还有一些方法使用**分层方法**或像**决策树**一样构建计划，在最终确定计划之前考虑所有可能的选项。虽然基于 LLM 的 agents 通常知识渊博，但它们有时在需要专业知识的任务中挣扎。将这些 agents 与特定领域的规划器集成已被证明可以提高它们的性能。

#### 计划反思（Plan Reflection）

创建计划后，agents 审查和评估其有效性非常重要。基于 LLM 的 agents 使用内部反馈机制，利用现有模型来完善它们的策略。它们还与人类交互，根据人类反馈和偏好调整它们的计划。Agents 还可以从真实和虚拟环境中收集见解，利用结果和观察来进一步完善它们的计划。

在规划中整合反馈的两种有效方法是 **ReAct** 和 **Reflexion**。

例如，**ReAct** 通过循环一系列**思想、行动和观察**来帮助 LLM 解决复杂任务，根据需要重复这些步骤。它从环境中接收反馈，这可能包括观察以及来自人类或其他模型的输入。这种方法允许 LLM 基于实时反馈调整其方法，增强其更有效地回答问题的能力。

---

### 4. Tools（工具）

在这个术语中，工具是各种资源，帮助 LLM agents 与外部环境连接以执行某些任务。这些任务可能包括从数据库中提取信息、查询、编码以及 agent 需要运行的任何其他内容。

当 LLM agent 使用这些工具时，它遵循特定的工作流程来执行任务、收集观察结果或收集完成子任务和满足用户请求所需的信息。

#### 工具集成示例

**MRKL（模块化推理、知识和语言）**：
此系统使用一组专家模块，从神经网络到计算器或天气 API 等简单工具。主要 LLM 充当**路由器**，根据任务将查询定向到适当的专家模块。

在一项测试中，LLM 被训练使用计算器来解决算术问题。研究发现，虽然 LLM 可以处理直接的数学查询，但在需要从文本中提取数字和运算的单词问题上挣扎。这突显了知道何时以及如何有效使用外部工具的重要性。

**Toolformer 和 TALM（工具增强语言模型）**：
这些模型专门针对与外部 API 有效交互进行了微调。例如，模型可以被训练使用金融 API 来分析股市趋势或预测货币波动，允许它直接向用户提供实时金融见解。

**HuggingGPT**：
此框架使用 ChatGPT 通过选择 HuggingFace 平台上可用的最佳模型来处理特定请求，然后总结结果来管理任务。

**API-Bank**：
一个基准测试，测试 LLM 使用 53 个常用 API 处理调度、健康数据管理或智能家居控制等任务的能力。

## LLM Agents 能做什么？

LLM agents 可以解决高级问题、从错误中学习、使用专门的工具来改进它们的工作，甚至与其他 agents 协作以提高它们的性能。以下是使 LLM agents 如此有价值的一些突出能力的更仔细观察：

### 1. 高级问题解决
LLM agents 可以高效地处理和执行复杂任务。它们可以生成项目计划、编写代码、运行基准测试、创建摘要等。这些任务展示了它们规划和执行需要高度认知参与的任务的能力。

### 2. 自我反思和改进
LLM agents 能够分析它们自己的输出，识别任何问题并进行必要的改进。这种自我反思能力使它们能够参与批评和重写的循环，在各种任务中持续提高它们的性能，例如编码、撰写文本和回答复杂问题。

### 3. 工具使用
LLM agents 可以评估它们自己的输出，确保工作的准确性和正确性。例如，它们可能会对它们的代码运行单元测试或使用网络搜索来验证文本中信息的准确性。这种关键评估帮助它们识别错误并提出必要的更正。

### 4. 多 Agent 框架
在多 agent LLM 框架中，一个 agent 可以生成输出，另一个可以批评并提供反馈，从而导致高级性能。

让我们看看一些著名的 LLM agents 和框架：

### LangChain
一个用于开发由 LLM 驱动的应用程序的框架，简化了 LLM 应用程序生命周期。
- CSV Agent
- JSON Agent
- OpenAPI Agent
- Pandas Dataframe Agent
- Python Agent
- SQL Database Agent
- Vectorstore Agent

### Llama Index
一个数据框架，通过数据连接器和结构化、高级检索接口和集成能力简化了 LLM 应用程序的创建。
- **Llama Hub**：社区驱动的数据加载器、读取器和工具库。

### Haystack
一个端到端的 NLP 框架，使你能够构建 NLP 应用程序。
- Haystack Agent
- SearchEngine
- TopPSampler

### Embedchain
一个为你的数据集创建类似 ChatGPT 的机器人的框架。
- JS Repo

### MindSearch
一个新的 AI 搜索引擎框架，工作方式类似于 Perplexity.ai Pro。你可以使用专有 LLM（如 GPT 和 Claude）或开源模型（如 InternLM2.5-7b-chat）将其设置为你自己的搜索引擎。它旨在浏览数百个网页来回答任何问题，提供详细的响应并显示它如何找到这些答案。

### AgentQ
帮助创建自主 web agents，可以规划、适应和自我纠正。它集成了引导蒙特卡洛树搜索（MCTS）、AI 自我批评和使用直接偏好优化（DPO）算法的 RLHF。

### Nvidia NIM Agent Blueprints
一个面向需要构建和部署定制 GenAI 应用程序的企业开发人员的 agent。

### Bee Agent Framework
IBM 的一个开源框架，用于大规模构建、部署和服务大型 agent 工作流程。IBM 的目标是通过 Bee 赋能开发者采用最新的开源和专有模型，同时对他们当前的 agent 实现进行最少的更改。

## LLM Agent 面临的挑战

虽然 LLM agents 非常有用，但它们确实面临几个我们需要考虑的挑战：

### 1. 有限的上下文
LLM agents 一次只能跟踪有限的信息量。这意味着它们可能不会记住对话早期的重要细节或错过关键指令。虽然向量存储等技术通过提供对更多信息有所帮助，但它们不能完全解决这个问题。

### 2. 长期规划的困难
LLM agents 很难制定跨越长时间的计划。当意外问题出现时，它们经常难以适应，这可能会使它们与人类解决问题的方式相比不那么灵活。

### 3. 不一致的输出
由于 LLM agents 依赖自然语言与其他工具和数据库交互，它们有时会产生不可靠的输出。它们可能会犯格式错误或不正确地遵循指令，这可能导致它们执行的任务中出现错误。

### 4. 适应特定角色
LLM agents 需要能够根据手头的任务处理不同的角色。然而，微调它们以理解和执行不常见的角色或与不同的人类价值观对齐是一个复杂的挑战。

### 5. 提示依赖
LLM agents 基于提示操作，但这些提示需要非常精确。即使是小的变化也可能导致大错误，因此创建和完善这些提示可能是一个微妙的过程。

### 6. 知识管理
保持 LLM agent 的知识准确和无偏见是棘手的。它们必须拥有正确的信息来做出明智的决策，但太多不相关的信息可能导致它们得出错误的结论或根据过时的事实采取行动。

### 7. 成本和效率
运行 LLM agents 可能是资源密集型的。它们通常需要快速处理大量数据，这可能是昂贵的，如果管理不当，可能会降低它们的性能。

解决这些挑战对于提高 LLM agents 在各种应用中的有效性和可靠性至关重要。

## 结论

总之，LLM agents 是解决复杂 LLM 任务的强大工具。它们可以规划、查找信息、记住过去的交互并从中学习，使它们在答案不是非黑即白时不可或缺。然而，它们有局限性，如短期记忆和对精确方向的需求。通过努力克服这些挑战，我们可以增强它们的能力，使它们在复杂 LLM 问题中更加有效和熟练。

## 常见问题解答

### 什么是 LLM agents？
LLM agents 是先进的 AI 系统，专为创建需要顺序推理的复杂文本而设计。它们能够提前思考、记住过去的对话，并使用不同的工具来根据所需的情境和风格调整它们的响应。

### LLM agent 的核心组件是什么？
LLM agents 通常由四个组件组成：agent 或大脑（是基于训练数据处理和理解语言的核心语言模型）、规划（能够将复杂的任务分解为可管理的子任务）、记忆（包括跟踪正在进行讨论的短期记忆和保留过去信息的长期记忆）以及工具使用（利用外部工具或数据库增强响应的能力）。

### LLM agents 与传统 LLM 或 RAG 系统有何不同？
虽然传统的 LLM 或检索增强生成（RAG）系统可以从数据库中获取信息，但它们通常缺乏将法律与实际业务情况联系起来或深入分析法院决策的能力。然而，LLM agents 可以将任务分解为子任务、建立历史基线并根据观察到的模式预测未来趋势。

### LLM agents 能做什么？
LLM agents 可以通过处理需要高度认知参与的复杂、多步骤任务（如生成项目计划、编写代码、运行基准测试和创建摘要）来解决高级问题。它们通过自我反思来提高性能，分析自己的输出以识别问题并进行必要的修订。通过使用单元测试或网络搜索等专业工具，它们确保工作的准确性和正确性。此外，在多 agent 框架中，LLM agents 可以通过生成输出和相互提供反馈进行协作，从而导致更先进和可靠的性能。

### 什么是 LLM agent 框架？
LLM agent 框架是用于构建和管理具有规划、记忆和工具使用等能力的 agents 的结构化系统。一些流行的框架包括 LangChain、AutoGen、CrewAI、MetaGPT 和 Superagent。这些框架提供了将 LLM agents 运营化以用于实际现实应用的基础设施。

### LLM agents 面临哪些挑战？
LLM agents 面临的挑战包括有限的上下文（限制了一次可以跟踪多少信息）、长期规划和适应意外问题的困难。由于依赖自然语言，它们可能会产生不一致的输出，并且需要非常精确的提示来避免错误。管理准确的知识是复杂的，运行这些 agents 可能是昂贵和资源密集型的。
