2026 年 OpenAI 真实内部实践

人类零行手动代码，
Codex 独立构建 100 万行产品

这不是 AI 辅助编程，而是彻底的智能体优先（Agent-First）开发范式。
人类从“写代码的人”转变为“设计 AI 高效工作环境（Harness）的人”。

01 实验起点与惊人成果

2025 年 8 月，从一个空的 Git 仓库开始，所有代码、架构、测试、文档全部由 Codex 生成，人类零行手动编写。

团队从 3 人扩大到 7 人，人均生产力反而显著提升。

02 人类角色彻底转变

工程师不再亲自写代码，而是：

  • 拆解产品目标
  • 设计工具与抽象层
  • 定义意图与反馈回路
  • 构建让 Codex 能可靠工作的“Harness”（缰绳系统）

“纪律仍然存在，只是位置变了——从代码层面转移到了支撑结构（工具、规范、反馈）。”

03 智能体可读性革命

最大瓶颈不再是模型能力，而是环境对 AI 是否可读。

  • 📸
    Chrome DevTools 协议：AI 可直接截图、操作 UI、复现 bug
  • 📊
    可观测性栈：LogQL 查询日志 + PromQL 查询指标

成果：单个 Codex 任务可连续自主运行 6 小时以上，常在人类睡觉时完成工作。

04 文档 = 地图而非百科全书

放弃又长又乱的 AGENTS.md，改为：

AGENTS.md（仅 ~100 行目录）
docs/
　├── design-docs/
　├── exec-plans/
　├── product-specs/
　└── ARCHITECTURE.md

给 AI 一张清晰的地图，而不是 1000 页容易过时的说明书。

05 架构必须机械强制

严格分层架构（Types → Config → Repo → Service → Runtime → UI），通过自定义 linter + 结构测试自动检查执行。

06 高吞吐量下的新流程

PR 极快迭代，审查逐步变为 agent-to-agent，人类仅在关键决策时介入。

07 Ralph Wiggum 循环

AI 自己生成代码 → 自己审查 → 响应反馈 → 反复迭代，直到所有 AI 审查员通过。

08 最大挑战：熵与垃圾收集

AI 容易复制坏模式，导致代码逐渐腐烂。必须建立后台“垃圾收集”智能体，定期扫描并发起清理 PR。

核心学习要点

1

Harness 的本质

不是更好的 Prompt，而是构建让 AI 可靠自主工作的完整环境（工具、可读性、强制规则、反馈回路）。

2

可读性是核心竞争力

UI、日志、指标、架构、文档都必须专门为智能体优化。

3

持续垃圾收集机制

AI 会制造熵，必须有专用智能体定期清理漂移代码。

立即可以做的行动清单

  1. 把你的 AGENTS.md 改成简短目录 + docs/ 结构化文件夹
  2. 定义 3–5 条核心架构规则，并尝试用 linter 强制执行
  3. 让 AI 更容易访问日志和界面（即使先从截图开始）
  4. 建立每周一次的“代码清理 PR”机制
