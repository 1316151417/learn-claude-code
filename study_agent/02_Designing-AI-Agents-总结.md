# 设计 AI Agents：核心知识总结

> 视频链接：https://www.youtube.com/watch?v=KHMwIsxqJSE
> 核心主题：AI Agent 架构的战略选择

---

## 核心观点

**架构不是可选项，而是战略必要性。**

选择正确的架构是区分成功与失败、可维护与调试噩梦、盈利与巨额成本的关键。

---

## 一、六项评估指标

| 指标 | 问题 |
|------|------|
| **推理能力** | Agent 如何思考？逐步还是一次性？ |
| **成本** | LLM 调用次数和 token 使用量 |
| **可靠性** | 能否处理意外路径？ |
| **延迟** | 响应速度如何？ |
| **可扩展性** | 能否从 1 个任务扩展到数百万个？ |
| **可解释性** | 能否理解决策原因？（审计/合规） |

## 二、四种核心架构

### 架构决策矩阵

```
┌─────────────────────────────────────────────────────────────────┐
│                      架构选择决策图                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  环境动态且不可预测？                                            │
│       │                                                        │
│       ├── YES → 需要实时适应？ → YES → 【ReAct】               │
│       │              ↓                                          │
│       │             NO                                           │
│       │                                                        │
│       └── NO → 问题结构良好可分解？                              │
│                │                                                │
│                ├── YES → 成本最小化？ → YES → 【计划然后执行】  │
│                │         ↓                                      │
│                │        NO                                       │
│                │                                                │
│                └── NO → 需要处理极端复杂性？ → YES → 【分层】   │
│                         │                                        │
│                         └── 需要长期运行+人工介入？ → 【图编排】 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1️⃣ ReAct（即兴者）

**原理**：推理 + 行动的持续循环

```
Thought → Action → Observation → Thought → ...
```

**优势**
- 灵活性最高
- 自然适应变化
- 适合动态环境

**劣势**
- 成本线性增长（每步都调用 LLM）
- 每天处理 100 万个任务（8 次调用/任务）≈ $40 万/天

**最佳场景**
- 客户支持聊天机器人
- 实时交互系统
- 探索性研究任务

---

### 2️⃣ 计划然后执行（蓝图）

**原理**：分工协作

```
规划者（昂贵模型，如 GPT-4）
    ↓ 输出详细计划
执行者（便宜模型，如 Haiku）
    ↓ 逐步执行
```

**优势**
- 成本效率高（昂贵模型只调用一次）
- 有清晰审计跟踪
- 模型能力最大化利用

**劣势**
- 系统脆弱（中途失败则计划作废）
- 重新规划成本高

**最佳场景**
- 结构良好的问题
- 成本敏感应用
- 需要审计跟踪的合规任务

---

### 3️⃣ 分层 Agents（专家团队）

**原理**：专业化分工 + 并行化

```
Manager Agent
    ├── Research Specialist
    ├── Data Analyst Specialist
    └── Writer Specialist
    ↓ 合成结果
```

**优势**
- 专业化（每个 agent 可独立优化）
- 并行处理（降低延迟）
- 可扩展性强

**劣势**
- 协调开销大
- agent 间通信复杂
- 调试难度高

**最佳场景**
- 企业级复杂系统
- 有明确领域分离的任务
- 需要并行化的场景

---

### 4️⃣ 图编排（迷宫）

**原理**：有限状态机

```
状态（State）+ 节点（Node）+ 边（Edge）
```

**优势**
- 可靠性最高
- 可恢复（中断后可从任意点重启）
- 天然支持人工介入

**劣势**
- 需要预先定义所有状态
- 灵活性差
- 架构师负担重

**最佳场景**
- 长期运行的流程
- 需要人工审批的工作流
- 高风险生产系统（如金融交易）

## 四、框架选择

### 工程师工具包（代码优先）

| 框架 | 特点 | 最适合 |
|------|------|-------|
| **OpenAI SDK** | 极简、4 个原语 | 快速原型、简单 ReAct |
| **LangGraph** | 综合生态、图支持 | 复杂生产系统 |
| **AutoGen** | 对话式多 agent | 多 agent 研究 |

### 企业工具包（结构化）

| 框架 | 特点 | 最适合 |
|------|------|-------|
| **CrewAI** | 角色化、易上手 | 基于角色的团队协作 |
| **Semantic Kernel** | 企业级、Azure 集成 | 微软生态企业应用 |
| **n8n** | 可视化、无代码 | 非技术人员、人工介入流程 |

## 五、实战场景匹配

| 场景 | 架构选择 | 框架推荐 |
|------|---------|---------|
| 客户支持聊天 | ReAct | OpenAI SDK / CrewAI |
| 数据分析管道 | 计划执行 + 图 | LangGraph / Semantic Kernel |
| 研究团队模拟 | 分层 | AutoGen / CrewAI |
| 保险索赔工作流 | 图编排 | n8n |

## 四大战略权衡

```
确定性    ←→    灵活性
   │                  │
延迟    ←→    推理深度
   │                  │
成本      ←→    质量
   │                  │
控制      ←→    自主性
```

**每个项目都必须在这四个维度上做出选择。**
