## 一、什么是 LLM Agents？

**定义**：LLM Agents 是先进的 AI 系统，专为创建需要**顺序推理**的复杂文本而设计。

### 与 RAG 的区别

| 特性 | RAG 系统 | LLM Agents |
|------|---------|------------|
| **能力** | 检索信息 | 检索 + 分解任务 + 建立基线 + 预测趋势 |
| **推理** | 有限 | 深度顺序推理 |
| **规划** | 无 | 有结构化规划 |
| **记忆** | 短期上下文 | 短期 + 长期记忆 |
| **工具** | 基本集成 | 深度工具集成 |

### 适用场景

```
简单查询（RAG 足够）
    │
    │ 复杂度高？
    ▼
需要顺序推理 + 规划 + 记忆
    │
    ▼
使用 LLM Agent
```

**示例**：
- ❌ "加州违约的法律后果是什么？" → RAG 足够
- ✅ "新隐私法下公司面临哪些挑战，法院如何处理？" → 需要 LLM Agent

## 二、四大核心组件

```
┌─────────────────────────────────────────────────────────────────┐
│                     LLM Agent 架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐  │
│  │ Agent/Brain  │      │   Memory     │      │   Planning   │  │
│  │  (语言模型)   │◄────►│  (记忆系统)   │◄────►│   (规划能力)  │  │
│  │              │      │  短期+长期   │      │  制定+反思   │  │
│  └──────────────┘      └──────────────┘      └──────────────┘  │
│         │                                                    │  │
│         └─────────────────────────────────────────────────┘   │
│                              │                                 │
│                              ▼                                 │
│                      ┌──────────────┐                          │
│                      │    Tools     │                          │
│                      │   (工具集)    │                          │
│                      │ MRKL/API/搜索 │                          │
│                      └──────────────┘                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1. Agent/Brain（大脑）
- 核心：语言模型（LLM/LAM）
- 输入：Prompt（指导如何响应、使用什么工具）
- 自定义：Persona（角色设定）

### 2. Memory（记忆）
| 类型 | 类比 | 特点 |
|------|------|------|
| **短期记忆** | 记事本 | 临时存储对话上下文，任务完成后清除 |
| **长期记忆** | 日记 | 存储数周/数月的见解，学习模式 |

### 3. Planning（规划）
**两阶段流程**：

```
┌─────────────────┐      ┌─────────────────┐
│  计划制定       │ ───► │  计划反思       │
│  Plan Formulation│      │  Plan Reflection│
└─────────────────┘      └─────────────────┘
        │                        │
        ▼                        ▼
┌─────────────────┐      ┌─────────────────┐
│ • 任务分解      │      │ • 内部反馈      │
│ • CoT/ToT       │      │ • 人类反馈      │
│ • 分层方法      │      │ • 环境观察      │
│ • 决策树        │      │ • ReAct/Reflexion│
└─────────────────┘      └─────────────────┘
```

**规划方法**：
- **CoT（思维链）**：逐个子任务解决，灵活自适应
- **ToT（思维树）**：探索多条路径，树状分支
- **分层方法**：考虑所有选项后决策

**反思机制**：
- **ReAct**：思想 → 行动 → 观察 循环
- **Reflexion**：基于反馈自我完善

### 4. Tools（工具）
**作用**：连接外部环境执行任务

| 工具系统 | 功能 |
|---------|------|
| **MRKL** | 模块化专家系统，LLM 作为路由器 |
| **Toolformer/TALM** | 微调模型与 API 交互 |
| **HuggingGPT** | 选择最佳模型处理任务 |
| **API-Bank** | 53 个常用 API 基准测试 |

## 三、LLM Agents 的四大能力

### 1. 高级问题解决
```
复杂任务
    ├── 生成项目计划
    ├── 编写代码
    ├── 运行基准测试
    └── 创建摘要
```

### 2. 自我反思和改进
```
输出 → 分析问题 → 批评 → 重写 → 改进
    ↑____________________________|
           持续循环
```

### 3. 工具使用验证
- 运行单元测试验证代码
- 使用搜索验证信息准确性

### 4. 多 Agent 协作
```
Agent A (生成) ────┐
                   ├──► 高级性能
Agent B (批评) ────┘
```

## 六、决策指南

### 何时使用 LLM Agent？

```
┌─────────────────────────────────────────┐
│         问题复杂度判断                   │
├─────────────────────────────────────────┤
│                                         │
│  简单事实查询？                          │
│       │                                │
│       ├── YES → 使用 RAG                │
│       │                                │
│       └── NO → 需要多步推理？           │
│                 │                       │
│                 ├── YES → 需要规划？    │
│                 │        │              │
│                 │        ├── YES → LLM Agent │
│                 │        └── NO → 考虑其他  │
│                 │                       │
│                 └── NO → 考虑其他方案    │
│                                         │
└─────────────────────────────────────────┘
```

### 核心判断标准

| 标准 | 说明 |
|------|------|
| **顺序推理** | 任务需要多步逻辑推理 |
| **规划需求** | 需要分解和计划子任务 |
| **记忆需求** | 需要记住之前的交互 |
| **工具使用** | 需要与外部系统交互 |

## 七、关键术语表

| 术语 | 解释 |
|------|------|
| **CoT** | Chain of Thought，思维链推理 |
| **ToT** | Tree of Thought，思维树探索 |
| **ReAct** | Reasoning + Acting，推理行动循环 |
| **Reflexion** | 基于反馈的自我反思机制 |
| **MRKL** | 模块化推理、知识和语言系统 |
| **RAG** | Retrieval Augmented Generation，检索增强生成 |
| **TALM** | Tool Augmented Language Models，工具增强语言模型 |
