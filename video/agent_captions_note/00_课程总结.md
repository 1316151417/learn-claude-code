## 四大设计模式回顾

### 1. 反射（Reflection）

**核心**：让AI审查并改进自己的输出

```
初稿 → 反思 → 改进 → 最终版本
```

**何时使用**：
- 需要高质量输出的场景
- 可以承受多次API调用的成本
- 有明确的评估标准

**关键技巧**：
- 使用推理模型做反思者效果更好
- 外部反馈（代码执行、事实核查）能显著提升效果
- 多个二元判断标准比单一评分更稳定

---

### 2. 工具使用（Tool Use）

**核心**：赋予LLM调用外部功能的能力

```
用户输入 → LLM决定 → 调用工具 → 处理结果 → 生成回答
```

**何时使用**：
- 需要最新信息（网络搜索）
- 需要执行计算（代码执行）
- 需要访问私有数据（数据库查询）

**关键技巧**：
- 清晰的工具描述和参数定义
- MCP标准化接口降低集成成本
- 工具选择要考虑成本和性能

---

### 3. 规划（Planning）

**核心**：LLM自主决定执行步骤的序列

```
理解任务 → 制定计划 → 执行计划 → 动态调整
```

**何时使用**：
- 开放性、探索性任务
- 步骤不确定的场景
- 需要灵活适应的情况

**关键技巧**：
- 规划 + 代码执行效果更强
- 高自主性可能带来惊艳结果，但降低可控性
- 根据场景选择合适的自主程度

---

### 4. 多智能体（Multi-Agent）

**核心**：多个智能体协同工作完成复杂任务

```
用户请求 → 协调器分配 → 各角色执行 → 整合结果
```

**何时使用**：
- 需要多种专业技能的任务
- 大型复杂项目
- 可以分解为独立子任务的工作

**关键技巧**：
- 明确每个角色的职责
- 选择合适的通信模式（顺序/并行/层级）
- 多智能体系统在复杂任务中表现更好

## 核心能力框架

### 任务分解能力

> 🎯 **最重要的技能之一**

```
复杂任务
   │
   ├─ 能否由LLM直接处理？
   │
   ├─ 能否通过简短代码完成？
   │
   └─ 是否需要调用工具？
```

### 评估与优化能力

```
构建 → 评估 → 分析 → 优化 → 迭代
```

**关键要点**：
- 能够驱动严格评估流程是成功构建工作流的最大预测因素
- 误差分析要检查中间输出，而不仅仅是最终结果
- 从10-15个测试用例开始建立评估数据集

## 结语

> 🎯 **核心观点**

"模型是80%，代码是20%。现代智能体工作的原因是模型被训练成智能体。代码的工作是提供工具，然后不要挡路。"

**记住**：
1. **迭代胜于一次性**：多步骤工作流显著提升质量
2. **评估驱动改进**：没有评估就无法优化
3. **模式可以组合**：反射、工具、规划、多智能体灵活搭配
4. **从简单开始**：不要一开始就追求复杂
5. **持续迭代**：智能体开发是循环改进的过程
