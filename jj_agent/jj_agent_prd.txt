基金分析 Agent 需求文档（PRD）

1. 背景（Background）

1.1 用户背景

目标用户为非专业但具备理性认知的长期基金投资者，核心特征：
	•	以指数基金 / ETF / 联接基金为主要投资工具
	•	以定投（每日 / 每周 / 每月）为主要执行方式
	•	投资周期 3–10 年
	•	不具备、也不希望具备高频盯盘与复杂分析能力

1.2 当前痛点
	1.	信息碎片化：
	•	基金数据分散在多个平台（估值、净值、宏观、新闻、情绪）
	•	需要频繁切换 App / 网站
	2.	分析门槛高：
	•	指标多（PE、PB、分位、盈利、成交额等）
	•	难以判断哪些“真的重要”
	3.	决策焦虑：
	•	不确定是否该加仓 / 停投 / 减持
	•	害怕买在“看起来像高点”的位置
	4.	时间与精力受限：
	•	无法每天跟踪市场
	•	需要快速、低频、可信的判断辅助

1.3 为什么需要 Agent

传统工具只提供“数据展示”，而非：
	•	状态判断
	•	阶段识别
	•	历史对比
	•	风险提示

本 Agent 的目标不是预测市场，而是：

系统性降低长期基金投资的认知负担与决策失误概率。

⸻

2. 设计目标（Design Goals）

2.1 核心目标

构建一个低频、稳健、可解释、可复盘的基金分析 Agent，用于：
	•	定期（每日 / 周 / 月）自动获取关键信息
	•	对基金与市场状态进行结构化分析
	•	辅助用户做“是否需要动作”的判断

2.2 非目标（明确不做）
	•	❌ 不做短线交易预测
	•	❌ 不做个股级别择时
	•	❌ 不给出强制买卖指令
	•	❌ 不追求 Alpha 或超额收益

2.3 设计原则
	1.	规则优先，AI 辅助
	2.	可解释 > 聪明
	3.	稳定一致 > 灵活多变
	4.	长期视角 > 短期噪音

⸻

3. 整体概要设计（High-Level Design）

3.1 系统分层架构

┌──────────────────────────┐
│        展示 / 通知层      │  → 文本摘要 / 周报 / 提醒
├──────────────────────────┤
│     AI 解释与分析层       │  → 阶段解读 / 风险提示
├──────────────────────────┤
│        规则引擎层         │  → 加仓 / 停投 / 观察
├──────────────────────────┤
│        数据处理层         │  → 清洗 / 归一化 / 存储
├──────────────────────────┤
│        数据采集层         │  → 指数 / 基金 / 新闻 / 情绪
└──────────────────────────┘

3.2 决策权与解释权分离
	•	规则引擎：拥有“判断权”（确定性）
	•	AI Agent：拥有“解释权”（辅助理解）

⸻

4. 核心能力清单（Detailed Capabilities）

4.1 基金与指数基础信息
	•	基金类型（ETF / 联接 / 主动）
	•	跟踪指数
	•	成立时间
	•	管理人 / 基金公司
	•	费用结构（管理费 / 托管费 / 申赎费）

4.2 核心量化指标（结构化）

4.2.1 估值类
	•	PE（TTM / 静态）
	•	PB
	•	股息率
	•	历史分位（5年 / 10年）

4.2.2 价格与趋势
	•	当前点位相对位置（1年 / 3年 / 5年）
	•	最大回撤
	•	波动率

4.2.3 基本面代理指标（指数级）
	•	盈利同比趋势
	•	ROE 区间
	•	成分股盈利覆盖率

4.3 市场状态识别
	•	阶段标签：
	•	极低估 / 偏低 / 合理 / 偏高 / 极高
	•	市场环境：
	•	流动性宽松 / 中性 / 收紧
	•	波动环境：
	•	低波动 / 高波动

4.4 新闻与事件摘要
	•	宏观政策（财政 / 货币）
	•	行业政策
	•	指数成分重大变化

要求：
	•	仅摘要与长期相关的信息
	•	过滤短期噪音与标题党

4.5 市场情绪指标（简化）
	•	成交额变化
	•	新增资金方向
	•	极端情绪信号（恐慌 / 狂热）

⸻

5. 决策辅助逻辑（规则引擎）

5.1 定投状态判断

状态	条件示例	建议动作
正常定投	估值合理	继续
放缓	高估 + 高波动	减少频率
可加仓	低估 + 恐慌	可选加仓
观察	信号冲突	不动作

5.2 原则
	•	所有规则版本化、可回溯
	•	不随模型输出动态变化

⸻

6. AI Agent 系统提示词（System Prompt 示例）

你是一个长期基金投资分析助手。
你的职责是：
1. 解释当前基金/指数所处的市场阶段
2. 对比历史相似阶段的共性与差异
3. 提示潜在风险与不确定性

严格限制：
- 不给出买入、卖出、加仓、减仓等直接操作指令
- 不预测未来点位或涨跌幅
- 不使用情绪化或煽动性语言

输出要求：
- 使用通俗、克制、长期视角的语言
- 明确区分“事实”“推断”“不确定性”


⸻

7. 工具与数据范围（Tooling Scope）

7.1 数据源
	•	官方指数公司（如中证）
	•	基金公司公告
	•	交易所公开数据

7.2 工具类型
	•	定时任务（cron）
	•	数据抓取 / API
	•	数据库存储（历史时间序列）
	•	LLM API（仅用于解释）

⸻

8. 上下文与记忆管理（Context & Memory）

8.1 长期记忆（Persistent Memory）
	•	基金历史估值时间序列
	•	历史规则触发记录
	•	历史阶段标签

8.2 短期上下文
	•	最近一次分析结果
	•	当前周期变化

8.3 复盘能力
	•	支持回答：
	•	“上一次类似情况发生在什么时候？”
	•	“当时市场后来如何演变？”

⸻

9. 输出形式（User-Facing）
	•	每日：一句话状态（可选）
	•	每周：结构化周报
	•	每月：阶段复盘 + 是否需要调整策略

⸻

10. 成功标准（Success Metrics）
	•	用户是否减少频繁查看行情
	•	是否能长期执行定投计划
	•	是否理解自己为何不操作

⸻

11. 总结

本基金分析 Agent 的核心价值不在于“更聪明”，而在于：

在不确定的市场中，提供稳定、可解释、可复盘的长期判断辅助。

这是一个服务于“长期正确性”的系统，而非短期收益工具。
